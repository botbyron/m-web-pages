# m-web-pages

一个用于归档与发布多个独立 HTML 页面的仓库。  
本仓库不针对任何特定域名或网站，仅作为静态页面的集中存放与版本管理场所。

## 项目简介

`m-web-pages` 旨在为各种小型、独立的 HTML 页面提供一套简单的归档与发布方案。  
你可以将需要长期保存的页面、临时演示页、或是离线可用的工具页面放入其中，通过 GitHub Pages 或其他静态托管服务直接访问。

仓库本身不包含任何后端逻辑、数据库或动态内容，所有页面均为纯前端 HTML/CSS/JavaScript 文件。

## 目录说明

```
m-web-pages/
├── pages/          # 存放所有独立 HTML 页面，每个页面一个子目录或独立文件
├── assets/         # 共享资源文件，如 CSS、JS、图片等（可选）
├── index.html      # 仓库首页，默认展示页面列表或导航
└── README.md       # 本文件
```

- **pages/**：核心目录，每个页面可单独命名，便于识别与引用。
- **assets/**：建议将多个页面共用的样式、脚本或图片放置于此，避免重复。
- **index.html**：可自定义为页面索引或简单导航，方便浏览。

## 页面归档说明

- 每个页面应保持独立、自包含，推荐将所有依赖（样式、脚本、图片等）内嵌或引用自 `assets/` 目录。
- 页面命名建议使用英文小写与连字符，例如 `example-page.html`。
- 如需更新某个页面，直接修改对应文件并提交即可，历史版本可通过 Git 追溯。
- 本仓库不限制页面内容，但请遵守开源社区规范，避免包含违法、侵权或恶意内容。

## 维护说明

- **添加新页面**：在 `pages/` 下创建新文件（或子目录），更新 `index.html` 中的导航（如有）。
- **删除页面**：移除对应文件，并同步更新索引。
- **更新页面**：直接编辑文件并提交，建议在 commit message 中简要说明改动。
- **共享资源**：若多个页面使用相同资源，请优先放入 `assets/` 并统一引用。

欢迎通过 Issue 提出建议或报告问题。  
Pull Request 请确保页面功能正常、无外部依赖失效，并保持目录结构清晰。
